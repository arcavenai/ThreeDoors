# Story 23.4: Doors Command — CLI Three Doors Experience

**Status:** Draft
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.1

## User Story

As a CLI user or LLM agent,
I want a `threedoors doors` command that presents three randomly selected tasks,
So that I can experience the core Three Doors mechanic without the TUI.

## Acceptance Criteria

**Given** the CLI scaffold from Story 23.1 is in place
**When** the doors command is implemented
**Then:**

- AC1: `threedoors doors` displays 3 randomly selected tasks in human-readable format: door number, task ID (short), text, status, type, effort
- AC2: `threedoors doors --json` outputs JSON envelope with `data` as array of `{door: N, task: {...}}` objects and `metadata.total_available` and `metadata.selection_method`
- AC3: Selection uses existing `SelectDoors()` from `internal/core/door_selector.go` — no duplication of selection logic
- AC4: If fewer than 3 tasks are available, display as many as exist with a note
- AC5: `threedoors doors --pick N` (where N is 1, 2, or 3) selects the Nth door and marks it in-progress — for scripting/LLM use
- AC6: `--pick` outputs the selected task details (human or JSON depending on `--json` flag)
- AC7: Non-interactive by default — prints and exits. No prompts unless `--interactive` is used
- AC8: Exit code 0 on success, exit code 1 if no tasks available

## Technical Notes

- The `doors` command is the signature CLI command — equivalent of launching the TUI
- `SelectDoors` or `SelectDoorsWithMood` are pure functions in `internal/core` — call directly
- `--pick` combines door selection + status change in one command — useful for LLM workflows
- `--interactive` flag is deferred to Story 23.10
- Reference: `docs/research/cli-interface-design.md` sections 4, 9

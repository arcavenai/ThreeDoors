# Story 18.1: Headless TUI Test Harness with teatest

## Status: done

## Story

As a developer,
I want a headless TUI test harness using Bubbletea's `teatest` package,
So that I can write automated tests that interact with the full TUI without a real terminal.

## Acceptance Criteria

- [x] `teatest` package (`github.com/charmbracelet/x/exp/teatest`) added to `go.mod`
- [x] Tests can create a `teatest.NewTestModel` with the root TUI model
- [x] Tests can send key messages (`tea.KeyMsg`) to simulate user input
- [x] Tests can retrieve `FinalOutput` and `FinalModel` for assertions
- [x] `lipgloss.SetColorProfile(termenv.Ascii)` enforced for deterministic output
- [x] Test helper `NewTestApp(t *testing.T, opts ...TestOption) *teatest.TestModel` provided in `internal/tui/testhelpers_test.go`
- [x] Helper accepts options: `WithTermSize(w, h int)`, `WithTaskFile(path string)`, `WithProvider(p TaskProvider)`
- [x] At least 3 smoke tests: app launch, door display, quit
- [x] Tests use stdlib `testing` package only; table-driven for >2 cases; t.Helper() in helpers; t.Cleanup() for resources

## Technical Notes

- `teatest` creates a pseudo-TTY internally — no Docker needed for basic headless tests
- Fixed terminal size (`teatest.WithInitialTermSize(80, 24)`) ensures reproducible layout
- The harness wraps the existing `tui.NewMainModel()` constructor — no TUI code changes needed
- Test fixtures use `t.TempDir()` for task files and config
- ASCII color profile set via `lipgloss.SetColorProfile(termenv.Ascii)` and restored in t.Cleanup()
- Filesystem isolated via `tasks.SetHomeDir(t.TempDir())` to prevent test side effects

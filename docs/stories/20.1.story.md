# Story 20.1: Reminders JXA Scripts and CommandExecutor

**Status:** draft
**Epic:** 20 — Apple Reminders Integration
**Blocks:** Stories 20.2, 20.3, 20.4

## User Story

As a developer,
I want JXA scripts for reading, creating, updating, completing, and deleting reminders,
So that the RemindersProvider has a reliable access layer for Apple Reminders.

## Acceptance Criteria

**Given** the need for Apple Reminders access on macOS
**When** the JXA scripts and executor are implemented
**Then:**

- AC1: `CommandExecutor` interface in `internal/adapters/reminders/` with `Execute(ctx, script) (string, error)`
- AC2: `OSAScriptExecutor` implementation using `osascript -l JavaScript`
- AC3: JXA script to read incomplete reminders from specified lists as JSON array (id, name, body, dueDate, priority, completed, flagged, creationDate, modificationDate)
- AC4: JXA script to read all reminder list names as JSON array
- AC5: JXA script to complete a reminder by ID (`completed = true`)
- AC6: JXA script to create a new reminder with name, body, priority, list
- AC7: JXA script to delete a reminder by ID
- AC8: Unit tests with mock executor returning canned JSON responses
- AC9: Scripts use `JSON.stringify()` for structured output (no brittle text parsing)

## Technical Notes

- Reuse `CommandExecutor` pattern from Apple Notes adapter (`internal/adapters/applenotes/`)
- JXA scripts called via `osascript -l JavaScript -e "<script>"`
- JXA output is JSON — parse with `encoding/json`, no text splitting
- Date format: ISO 8601 via `toISOString()` in JXA
- Reminder IDs are `x-apple-reminder://` URIs — stable and persistent
- `whose` clause filtering: `list.reminders.whose({completed: false})()`
- Platform guard: `//go:build darwin` on executor implementation
- Store scripts as Go string constants in `jxa_scripts.go`
- Escape user input in JXA scripts to prevent injection (NFR-IS1)

# Story 23.6: Task Block, Unblock, and Status Commands

**Status:** Draft
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.2

## User Story

As a CLI user,
I want to block, unblock, and change task status from the command line,
So that I can manage task state transitions without the TUI.

## Acceptance Criteria

**Given** the task list/show commands from Story 23.2 are in place
**When** status management commands are implemented
**Then:**

- AC1: `threedoors task block <id> --reason "Waiting on API"` marks the task as blocked with a reason — supports prefix matching
- AC2: `threedoors task unblock <id>` transitions a blocked task back to `todo` — supports prefix matching
- AC3: `threedoors task status <id> <new-status>` changes task status to any valid status (in-progress, in-review, etc.)
- AC4: Invalid status transitions return exit code 3 with descriptive error: "invalid transition from <current> to <target>"
- AC5: `--reason` is required for `block` — error if omitted
- AC6: All commands support `--json` flag for structured output
- AC7: Human-readable output confirms the change: "Task <short-id> status: <old> -> <new>"
- AC8: Batch support for status changes: `threedoors task status <id1> <id2> in-progress`

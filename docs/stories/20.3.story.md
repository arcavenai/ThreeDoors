# Story 20.3: Reminders Write Support

**Status:** Done (PR #155)
**Epic:** 20 — Apple Reminders Integration

## User Story

As a ThreeDoors user,
I want to complete, create, and delete reminders from within ThreeDoors,
So that changes sync back to Apple Reminders on all my devices via iCloud.

## Acceptance Criteria

**Given** an Apple Reminders task is modified in ThreeDoors
**When** the write operation executes
**Then:**

- AC1: `MarkComplete(taskID)` sets `completed = true` via JXA script
- AC2: `SaveTask(task)` creates new reminder if ID is empty, updates existing if ID matches
- AC3: `DeleteTask(taskID)` removes reminder via JXA script
- AC4: Error categorization: permission denied, reminder not found, timeout (distinct error types)
- AC5: Retry logic with configurable attempts and exponential backoff (reuse pattern from applenotes)
- AC6: Full contract test compliance (stable IDs enable complete round-trip testing)
- AC7: SaveTask maps Task fields back to Reminder fields (Text → name, Notes → body, Effort → priority)

## Technical Notes

- Error types: define `ErrPermissionDenied`, `ErrReminderNotFound`, `ErrTimeout` in `errors.go`
- Reuse retry pattern from `internal/adapters/applenotes/retry.go`
- Reverse priority mapping: deep-work → 1, medium → 5, quick-win → 9
- SaveTask create: if task.ID is empty, create via JXA, then read back the new ID
- SaveTask update: locate reminder by ID, update fields
- MarkComplete: locate by ID, set completed = true
- Escape task text in JXA to prevent injection (quotes, backslashes, newlines)
- iCloud sync happens transparently — no need to trigger it

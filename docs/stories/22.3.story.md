# Story 22.3: TUI Dispatch Key Binding and Confirmation Flow

**Status:** draft
**Epic:** 22 ŌĆö Self-Driving Development Pipeline
**Depends on:** Story 22.1

## User Story

As a user,
I want to press 'x' in the task detail view or type `:dispatch` to dispatch a task to the dev queue,
So that I can trigger automated development work on a selected task.

## Acceptance Criteria

**Given** a task is selected in the detail view and dev dispatch is enabled
**When** the user presses 'x' or types `:dispatch`
**Then:**

- AC1: Confirmation dialog appears: "Dispatch '<task text>' to dev queue? [y/n]"
- AC2: On 'y', task is added to dev queue with status `pending` and `Task.DevDispatch.Queued` set to `true`, `QueuedAt` set to current UTC time
- AC3: On 'n', confirmation is dismissed with no side effects
- AC4: If task is already dispatched (`Task.DevDispatch != nil && Task.DevDispatch.Queued`), show message "Task already dispatched" and do not re-enqueue
- AC5: If multiclaude is not available (checked via `Dispatcher.CheckAvailable`), 'x' key and `:dispatch` command are hidden from key binding help and produce no action
- AC6: If `dev_dispatch_enabled` is `false` in config, 'x' key and `:dispatch` command are not registered at all
- AC7: `[DEV]` badge rendered on dispatched tasks in the doors view ŌĆö displayed next to the task status badge
- AC8: Successful dispatch shows brief confirmation message: "Task queued for dev dispatch"

## Technical Notes

- The confirmation dialog follows the same pattern as existing confirm dialogs in the TUI (e.g., task completion confirmation)
- `[DEV]` badge uses Lipgloss styling ŌĆö suggest a gear-like color (e.g., cyan or yellow)
- The `:dispatch` command integrates with the existing command palette infrastructure
- Key binding registration is conditional: only add 'x' handler if `config.DevDispatchEnabled && dispatcher.CheckAvailable() == nil`
- All user-visible output goes through Bubbletea `View()` ŌĆö no `fmt.Println`
- Queue write is wrapped in a `tea.Cmd` to avoid blocking the TUI update loop

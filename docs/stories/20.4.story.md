# Story 20.4: Reminders Config, Registration, and Health Check

**Status:** draft
**Epic:** 20 — Apple Reminders Integration

## User Story

As a ThreeDoors user,
I want to configure Apple Reminders in config.yaml with list filtering,
So that I only see reminders from my work-related lists.

## Acceptance Criteria

**Given** the Reminders adapter is implemented
**When** a user adds Reminders config to config.yaml
**Then:**

- AC1: Config section: `lists` (comma-separated list names), `include_completed` (bool, default false)
- AC2: Adapter factory wired to config parsing
- AC3: Registration in `RegisterBuiltinAdapters()` as `"reminders"`
- AC4: `HealthCheck()` returns clear guidance when Reminders access is denied in System Settings
- AC5: Platform guard: `//go:build darwin` — adapter only available on macOS
- AC6: On non-macOS platforms, factory returns descriptive error
- AC7: Unit tests for config parsing, validation, and platform guard

## Technical Notes

- Config follows existing pattern: `providers: [{name: reminders, settings: {lists: "Work,ThreeDoors"}}]`
- Empty lists setting = all lists (default behavior)
- HealthCheck message on permission denied: "Grant Reminders access in System Settings > Privacy & Security > Reminders"
- Platform guard on factory: use `//go:build darwin` file and a stub file for other platforms
- The stub file for non-darwin should register a factory that returns an error explaining macOS is required

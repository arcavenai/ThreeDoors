# Story 20.2: Reminders Read-Only Provider

**Status:** Done (PR pending)
**Epic:** 20 — Apple Reminders Integration
**Blocks:** Stories 20.3, 20.4

## User Story

As a ThreeDoors user on macOS,
I want my Apple Reminders to appear as tasks in ThreeDoors,
So that I can use Three Doors selection for my reminder lists.

## Acceptance Criteria

**Given** Apple Reminders contains incomplete reminders
**When** LoadTasks is called
**Then:**

- AC1: `RemindersProvider` in `internal/adapters/reminders/reminders_provider.go` implementing `core.TaskProvider`
- AC2: `LoadTasks()` reads incomplete reminders via JXA, parses JSON, maps to `[]*core.Task`
- AC3: Field mapping: id → ID, name → Text, body → Notes[0].Text, priority → Effort, completed → Status, dates → timestamps
- AC4: Priority mapping: 1-4 → deep-work, 5 → medium, 6-9 → quick-win, 0 → (empty)
- AC5: Configurable list filtering (empty = all lists)
- AC6: `SaveTask/SaveTasks/DeleteTask/MarkComplete` return `core.ErrReadOnly`
- AC7: `HealthCheck()` attempts lightweight read, reports TCC permission status
- AC8: `Name()` returns `"reminders"`
- AC9: Contract tests pass via `adapters.RunContractTests`
- AC10: Table-driven field mapping tests for all priority/status combinations

## Technical Notes

- Task.ID = reminder's persistent `x-apple-reminder://` URI (stable across sync/reorder)
- Task.SourceProvider = `"reminders:<list-name>"`
- Priority scale is inverted from typical: 1 = highest in Reminders
- Due date: encode in Context field for now (e.g., `"due:2026-03-10"`)
- HealthCheck detects TCC permission denial and returns helpful guidance
- Constructor: `NewRemindersProvider(executor CommandExecutor, lists []string) *RemindersProvider`
- Accept interfaces (CommandExecutor), return concrete type

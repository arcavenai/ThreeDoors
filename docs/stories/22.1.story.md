# Story 22.1: Dev Dispatch Data Model and Queue Persistence

**Status:** draft
**Epic:** 22 — Self-Driving Development Pipeline
**Blocks:** Stories 22.2, 22.3

## User Story

As a developer,
I want a `DevDispatch` struct on the `Task` type and a file-based dev queue,
So that dispatch state is tracked independently from task lifecycle and persists across TUI restarts.

## Acceptance Criteria

**Given** the need to track dev dispatch state orthogonal to task status
**When** the data model is created
**Then:**

- AC1: `DevDispatch` struct defined in `internal/dispatch/model.go` with fields: Queued (`bool`), QueuedAt (`*time.Time`), WorkerName (`string`), PRNumber (`int`), PRStatus (`string`), DispatchErr (`string`) — all with `yaml:"...,omitempty"` and `json:"...,omitempty"` tags
- AC2: `QueueItem` struct defined in `internal/dispatch/model.go` with fields: ID (`string`), TaskID (`string`), TaskText (`string`), Context (`string`), Status (`QueueItemStatus` — pending/dispatched/completed/failed), Priority (`int`), Scope (`string`), AcceptanceCriteria (`[]string`), QueuedAt/DispatchedAt/CompletedAt (`*time.Time`), WorkerName (`string`), PRNumber (`int`), PRURL (`string`), Error (`string`)
- AC3: `DevQueue` struct with methods: `Load(path string) error`, `Save(path string) error`, `Add(item QueueItem) error`, `Get(id string) (QueueItem, error)`, `Update(id string, fn func(*QueueItem)) error`, `List() []QueueItem`
- AC4: Queue file location defaults to `~/.threedoors/dev-queue.yaml`; configurable via constructor
- AC5: Queue persistence uses atomic write pattern (write to `.tmp`, fsync, rename) — consistent with existing file persistence patterns
- AC6: `Task` struct in `internal/core/task.go` extended with `DevDispatch *DevDispatch` field (pointer, `yaml:"dev_dispatch,omitempty"` `json:"dev_dispatch,omitempty"`)
- AC7: Unit tests for queue CRUD operations (add, get, update, list, not-found error) and atomic write safety (verify temp file cleanup)
- AC8: `QueueItemStatus` type with constants and `String()` method for serialization

## Technical Notes

- `DevDispatch` is a pointer field on `Task` to maintain zero-value usefulness — nil means "never dispatched"
- Queue ID format: `dq-` prefix + first 8 chars of UUID (e.g., `dq-a855835c`)
- `QueueItemStatus` is a string type, not iota, for YAML readability
- The `internal/dispatch/` package is new — follows project convention of one primary concern per package
- Import `internal/core` for the `Task` type extension, but keep dispatch logic in `internal/dispatch/`
- Timestamps use `time.Now().UTC()` per project conventions
- No `init()` functions — pass dependencies via constructors

# Story 19.1: Jira HTTP Client

**Status:** Done (PR #132)
**Epic:** 19 — Jira Integration
**Blocks:** Stories 19.2, 19.3, 19.4

## User Story

As a developer,
I want a thin HTTP client for the Jira REST API v3,
So that the JiraProvider can query and transition issues without a third-party SDK dependency.

## Technical Notes

- Use `net/http` + `encoding/json` — no third-party Jira SDK
- Auth config: `AuthConfig{Type, URL, Email, APIToken}` — Type is "basic" or "pat"
- Basic Auth: `Authorization: Basic base64(email:api_token)`
- PAT Bearer: `Authorization: Bearer <token>`
- The new search endpoint is `POST /rest/api/3/search/jql` (legacy `GET /rest/api/3/search` removed from Cloud)
- Pagination uses `nextPageToken` (no `startAt`); detect last page via `isLast: true`
- Response types: `SearchResult{Issues []Issue, NextPageToken string, IsLast bool}`
- Handle 409 Conflict on transitions (concurrent transition in progress)
- Accept interfaces, return concrete types: `NewClient(config AuthConfig) *Client`
- Context first parameter on all methods

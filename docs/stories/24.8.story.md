# Story 24.8: MCP Prompt Templates & Advanced Interaction Patterns

**Status:** draft
**Epic:** 24 — MCP/LLM Integration Server
**Depends on:** Stories 24.6, 24.7

## User Story

As an LLM client,
I want pre-built prompt templates for common interactions (deep dives, prioritization, workload analysis, burnout alerts, focus recommendations),
So that I can provide high-quality coaching and analysis with consistent, well-structured responses.

## Acceptance Criteria

**Given** analytics, graph, and query tools are available
**When** an LLM uses prompt templates for advanced interactions
**Then:**

- AC1: MCP Prompt `blocked_tasks` registered — guides LLM to query blocked tasks, show blockers, and suggest unblocking actions
- AC2: MCP Prompt `task_deep_dive` registered — guides LLM to get task detail, walk relationship graph (depth 2), check session history, and report full context with suggestions
- AC3: MCP Prompt `weekly_retrospective` registered — guides LLM to get weekly completions by day, mood data, pattern analysis, and narrate best day, trends, correlations, recommendations
- AC4: MCP Tool `prioritize_tasks(limit, mood, time_of_day)` — returns scored task list using signals: blocking score, age, effort fit, type fit, time-of-day fit, streak impact, provider urgency
- AC5: `PrioritySuggestion` struct: `TaskID`, `Score` (0-100), `Rationale`, `Factors []Factor` where `Factor` has `Name`, `Weight`, `Value`
- AC6: MCP Tool `analyze_workload()` — returns `WorkloadAnalysis`: total tasks, by status/effort/provider, estimated hours (from historical effort-to-time mapping), overload risk (0-1), recommended focus (top 3 IDs), defer candidates
- AC7: MCP Tool `focus_recommendation(mood, available_minutes)` — returns `FocusRecommendation`: optimal start time, duration, suggested tasks, task order, break timing, rationale
- AC8: MCP Tool `what_if(complete_task_ids)` — models scenario without mutation: unblocked tasks, streak impact, velocity change, epic progress, estimated time
- AC9: MCP Tool `context_switch_analysis(session_id)` — returns switches per session, cost per switch, most expensive type pairs, optimal batching suggestions
- AC10: Unit tests for prioritization scoring, workload analysis, and what-if modeling

## Technical Notes

- MCP Prompts are YAML template strings, not executable code — they guide LLMs to call the right tools in the right order
- Prioritization uses multi-signal scoring — each factor has a weight and value, final score is weighted sum normalized to 0-100
- `what_if` creates an in-memory copy of the task graph, applies hypothetical completions, and computes downstream effects — never mutates real data
- Workload estimation uses historical median time-per-effort-level from session data
- Focus recommendation combines time-of-day productivity profile with current mood and available time
- Context switch cost estimated from session data: time gap between task type transitions vs same-type sequences

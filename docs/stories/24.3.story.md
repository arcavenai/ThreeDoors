# Story 24.3: Security Middleware — Rate Limiting & Audit Logging

**Status:** Done (PR #179)
**Epic:** 24 — MCP/LLM Integration Server
**Depends on:** Story 24.1

## User Story

As a system administrator,
I want the MCP server to enforce rate limits, log all requests, and validate inputs,
So that the server is protected against abuse and maintains a tamper-evident audit trail.

## Acceptance Criteria

**Given** the MCP server is handling requests from LLM clients
**When** middleware is applied to the request pipeline
**Then:**

- AC1: `RateLimiter` middleware enforces: global 100 req/min per connection, 20 proposals/min, 60 queries/min per tool, max 5 pending proposals per task, 10-request burst allowance
- AC2: `AuditLogger` middleware logs every request to `~/.threedoors/mcp-audit.jsonl` with fields: `ts`, `req_id`, `tool`, `args`, `result` (ok/error/rate_limited), `source`, `duration_ms`, `error`, `prev_hash`
- AC3: Audit log entries include SHA-256 hash chain (`prev_hash` = SHA-256 of previous entry) for tamper detection
- AC4: `SchemaValidator` middleware validates inputs: task IDs match UUID v4 format, text fields max 500 chars, status values are valid `TaskStatus` enum, timestamps are valid UTC and not in the future
- AC5: `ReadOnlyEnforcer` middleware blocks any request that would directly call `TaskProvider.SaveTask()` — write operations only through `ProposalStore`
- AC6: Middleware chain applied in order: `ReadOnlyEnforcer` → `RateLimiter` → `AuditLogger` → `SchemaValidator` → `coreHandler`
- AC7: Rate limit violations return standard MCP error response with retry-after hint
- AC8: Audit log rotates daily with 30-day retention
- AC9: Unit tests for each middleware independently and for the composed chain

## Technical Notes

- Middleware follows the `func(Handler) Handler` pattern from Story 24.1
- Rate limiting uses `golang.org/x/time/rate` or equivalent token bucket
- Audit log follows the same JSONL pattern as existing `SyncLog`
- SHA-256 hash chain is lightweight — hash the JSON bytes of the previous entry
- `ReadOnlyEnforcer` is a compile-time safety net — the API surface should never expose writes, but this catches mistakes
- Daily rotation: filename format `mcp-audit-2026-03-07.jsonl`

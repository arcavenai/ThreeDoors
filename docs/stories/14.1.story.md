## Technical Design

### Spike Scope

This is a research + prototype spike. Deliverables are:

1. **Spike Report** (`docs/spikes/llm-decomposition.md`) — comprehensive analysis document
2. **Prototype Code** (`internal/intelligence/llm/`) — minimal working prototype demonstrating:
   - `LLMBackend` interface with pluggable provider pattern
   - `OllamaBackend` — local LLM via Ollama HTTP API
   - `ClaudeBackend` — cloud LLM via Anthropic API
   - `TaskDecomposer` — prompt engineering + structured output parsing
   - `GitOutputWriter` — write BMAD-style story specs to git repo

### LLMBackend Interface

```go
// LLMBackend abstracts LLM provider communication.
type LLMBackend interface {
    Name() string
    Complete(ctx context.Context, prompt string) (string, error)
    Available() bool
}
```

### Provider Implementations

**Ollama (local):**
- HTTP POST to `http://localhost:11434/api/generate`
- Configurable model name (default: `llama3.2`)
- No auth required, availability check via health endpoint

**Claude API (cloud):**
- HTTP POST to `https://api.anthropic.com/v1/messages`
- API key from config or env var `ANTHROPIC_API_KEY`
- Model: `claude-sonnet-4-20250514` (or configurable)

### Output Schema

Task decomposition outputs structured BMAD story specs:

```go
type StorySpec struct {
    Epic        string   `json:"epic"`
    StoryID     string   `json:"story_id"`
    Title       string   `json:"title"`
    UserStory   string   `json:"user_story"`
    ACs         []string `json:"acceptance_criteria"`
    Tasks       []string `json:"tasks"`
    DevNotes    string   `json:"dev_notes"`
}
```

### Git Output

- Write specs to configurable repo path
- Create branch, commit files, optionally create PR
- Uses `os/exec` for git operations (no go-git dependency in spike)

### Config Integration

```yaml
llm:
  backend: "ollama"  # or "claude"
  ollama:
    endpoint: "http://localhost:11434"
    model: "llama3.2"
  claude:
    model: "claude-sonnet-4-20250514"
    # API key from ANTHROPIC_API_KEY env var
  decomposition:
    output_repo: ""  # path to target git repo
    output_branch_prefix: "story/"
```

## Dev Notes

- Mock all external HTTP calls in tests — no real LLM calls in CI
- Follow existing project patterns: table-driven tests, error wrapping with `%w`
- The spike report is as important as the code — it drives the build/wait decision
- Use `net/http` stdlib for HTTP clients (per tech stack principles)
- Timestamps in UTC per project standards

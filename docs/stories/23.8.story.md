# Story 23.8: Mood and Stats Commands

**Status:** Done (PR #175)
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.1

## User Story

As a CLI user,
I want to record my mood and view productivity statistics from the command line,
So that I can track my patterns and well-being without the TUI.

## Acceptance Criteria

**Given** the CLI scaffold from Story 23.1 is in place
**When** mood and stats commands are implemented
**Then:**

- AC1: `threedoors mood set <mood>` records a mood (focused, energized, tired, stressed, neutral, etc.) via `SessionTracker.RecordMood()`
- AC2: `threedoors mood set custom "Feeling creative"` records a custom mood string
- AC3: `threedoors mood history` shows recent mood entries in human-readable format
- AC4: `threedoors mood history --json` outputs mood history as JSON envelope
- AC5: `threedoors stats` displays a summary dashboard: tasks completed today, streak, completion rate
- AC6: `threedoors stats --daily` shows daily completions for the last 7 days via `PatternAnalyzer.GetDailyCompletions()`
- AC7: `threedoors stats --weekly` shows week-over-week comparison via `PatternAnalyzer.GetWeekOverWeek()`
- AC8: `threedoors stats --patterns` shows full pattern analysis via `PatternAnalyzer.Analyze()`
- AC9: `threedoors stats --json` outputs all stats in JSON envelope
- AC10: All mood values validated — invalid mood returns exit code 3

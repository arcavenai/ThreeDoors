# Story 1.7: CI/CD Pipeline & Alpha Release

Status: Done

## Story

As a developer working in a multi-agent environment,
I want automated CI/CD quality gates and alpha release packaging,
so that every PR is validated for formatting, linting, tests, and build correctness, and merged PRs produce downloadable alpha binaries.

## Acceptance Criteria

1. GitHub Actions CI workflow file created at `.github/workflows/ci.yml`
2. **PR Quality Gate job** runs on every `pull_request` targeting `main`:
   - Checks out code with `actions/checkout@v6`
   - Sets up Go with `actions/setup-go@v6` using `go-version-file: 'go.mod'`
   - Runs `gofumpt -l .` and fails if any files need formatting
   - Runs `go vet ./...`
   - Runs `golangci-lint` via `golangci/golangci-lint-action@v6` with `version: v1.61.0`
   - Runs `go test ./... -v -count=1` (cache disabled in CI)
   - Runs `go build -o /dev/null ./cmd/threedoors` to validate build
3. **Alpha Release job** runs on `push` to `main` (i.e., PR merge):
   - Builds binaries for: `darwin/arm64`, `darwin/amd64`, `linux/amd64`
   - Uses `GOOS` and `GOARCH` environment variables for cross-compilation
   - Uploads binaries as workflow artifacts (not full GitHub Releases)
   - Artifact names include short SHA: `threedoors-<os>-<arch>-<short-sha>`
4. All jobs run on `ubuntu-latest` (GitHub-hosted public runners)
5. Go module caching enabled via `actions/setup-go@v6` built-in cache
6. Workflow passes on current codebase (all existing code must pass fmt, lint, vet, test, build)
7. If current codebase has formatting/linting issues, fix them as part of this story

### Pre-PR Submission Checklist

Before pushing your branch or creating a PR, complete every item:

- [ ] **Rebase onto latest main**: Run `git fetch upstream main && git rebase upstream/main` — resolve any conflicts before pushing (PRs #3, #5, #19, #23 were delayed by merge conflicts)
- [ ] **Run gofumpt**: Run `gofumpt -l .` and verify **no output** — if files are listed, run `gofumpt -w .` to fix (PRs #9, #10, #23, #24 required gofumpt fix-up commits)
- [ ] **Run golangci-lint**: Run `golangci-lint run ./...` and verify **0 issues** — pay attention to `errcheck` (unchecked error returns) and `staticcheck` (PR #16 needed 2 extra commits for lint fixes)
- [ ] **Run all tests**: Run `go test ./... -count=1` and verify **0 failures**
- [ ] **Check for dead code**: Run `go vet ./...` and review for unused functions, variables, or imports (PR #28 shipped unused `SortedDates()` method)
- [ ] **Verify no out-of-scope files**: Review `git diff --stat` — ensure only files related to this story are changed (PR #5 included an unrelated `agents/` directory)
- [ ] **Verify CI workflow locally**: After creating the workflow file, validate YAML syntax and ensure all referenced actions/versions exist
- [ ] **Single clean commit preferred**: Squash or amend fix-ups before pushing — avoid separate "fix: address code review findings" commits (pattern seen in ~15 PRs)

## Dev Notes

### Critical Implementation Details

**Workflow file structure:**
```yaml
name: CI
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  quality-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-go@v6
        with:
          go-version-file: 'go.mod'
      # fmt check, vet, lint, test, build steps

  alpha-release:
    needs: quality-gate
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      # cross-compile and upload artifacts
```

**gofumpt check pattern** (CI-friendly, does not modify files):
```bash
# List files needing formatting - fail if any output
UNFORMATTED=$(gofumpt -l .)
if [ -n "$UNFORMATTED" ]; then
  echo "Files need formatting:"
  echo "$UNFORMATTED"
  exit 1
fi
```

**gofumpt installation in CI (pin version per tech-stack.md):**
```bash
go install mvdan.cc/gofumpt@v0.7.0
```

**Cross-compilation pattern:**
```bash
GOOS=darwin GOARCH=arm64 go build -o threedoors-darwin-arm64 ./cmd/threedoors
GOOS=darwin GOARCH=amd64 go build -o threedoors-darwin-amd64 ./cmd/threedoors
GOOS=linux GOARCH=amd64 go build -o threedoors-linux-amd64 ./cmd/threedoors
```

**Artifact upload (14-day retention for alpha builds):**
```yaml
- uses: actions/upload-artifact@v4
  with:
    name: threedoors-binaries-${{ github.sha }}
    path: threedoors-*
    retention-days: 14
```

### golangci-lint Action

**Version conflict resolution:** The tech-stack.md references golangci-lint `1.61.0` (v1), but `golangci-lint-action@v7` requires golangci-lint v2+. Resolution: use `golangci-lint-action@v6` which supports v1, OR upgrade to golangci-lint v2 and update tech-stack.md. Prefer using `golangci-lint-action@v6` with `version: v1.61.0` to match documented tech stack — upgrading to v2 is out of scope for this story.

**No `.golangci.yml` exists.** Create a minimal config file:
```yaml
# .golangci.yml
run:
  timeout: 5m
linters:
  enable:
    - gofumpt
```

### CI Step Ordering (Fail-Fast)

Run steps in this exact order to fail fast on cheapest checks:
1. `gofumpt -l .` (formatting check - instant)
2. `go vet ./...` (correctness - fast)
3. `golangci-lint` (static analysis - medium)
4. `go test ./... -v -count=1` (tests - slowest, `-count=1` disables cache in CI)
5. `go build` (build validation)

### Artifact Retention

Set `retention-days: 14` on upload-artifact to avoid storage bloat. Alpha builds don't need 90-day default retention.

### What NOT to Do

- Do NOT use GoReleaser - it's overkill for this project. Simple `go build` cross-compilation is sufficient.
- Do NOT create full GitHub Releases with tags - use workflow artifacts only for alpha builds.
- Do NOT add Windows builds - not a target platform per architecture docs (macOS primary).
- Do NOT modify any Go source code unless it fails formatting/linting. This is an infrastructure-only story.
- Do NOT add branch protection rules via the workflow - that's a repo admin setting, just document the recommendation.

# Story 23.2: Task List and Task Show Commands with Prefix Matching

**Status:** Done
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.1

## User Story

As a CLI user,
I want to list tasks with filters and view task details by ID prefix,
So that I can browse and inspect my tasks without launching the TUI.

## Acceptance Criteria

**Given** the CLI scaffold from Story 23.1 is in place
**When** the task list and show commands are implemented
**Then:**

- AC1: `threedoors task list` displays all active tasks in a tabwriter table with columns: ID (short), STATUS, TYPE, EFFORT, TEXT
- AC2: `threedoors task list --status <status>` filters by task status (todo, in-progress, blocked, done, etc.)
- AC3: `threedoors task list --type <type>` filters by task type
- AC4: `threedoors task list --effort <effort>` filters by effort level
- AC5: Filters are composable: `--status todo --type code` applies both
- AC6: `threedoors task list --json` outputs JSON with the standard envelope, including `metadata.total`, `metadata.filtered`, and `metadata.filters`
- AC7: `threedoors task show <id>` displays full task detail (all fields) — supports ID prefix matching
- AC8: `threedoors task show <id> --json` outputs full task as JSON
- AC9: `FindByPrefix(prefix string)` method added to `TaskPool` — returns single match, `ErrTaskNotFound` for no matches, descriptive error for ambiguous prefixes
- AC10: Short IDs displayed in list output (first 8 characters of UUID)
- AC11: Footer shows count: "N tasks found" (human mode)
- AC12: Exit code 2 for not-found, exit code 5 for ambiguous prefix

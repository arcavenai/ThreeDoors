# Story 24.6: Pattern Mining & Mood-Execution Analytics

**Status:** draft
**Epic:** 24 — MCP/LLM Integration Server
**Depends on:** Story 24.2

## User Story

As an LLM client,
I want to access productivity analytics including mood-execution correlations, streaks, burnout risk, and time-of-day patterns,
So that I can provide data-driven productivity insights and coaching to the user.

## Acceptance Criteria

**Given** session history and mood data exist in ThreeDoors
**When** an LLM queries analytics endpoints
**Then:**

- AC1: `PatternMiner` struct in `internal/mcp/analytics.go` wrapping `SessionStore`, `TaskPool`, and `EnrichmentStore`
- AC2: `MoodCorrelation(from, to time.Time) (*MoodCorrelation, error)` — returns mood rating vs avg completions/session, avg session duration, and task type preferences per mood
- AC3: `ProductivityProfile(from, to time.Time) (*ProductivityProfile, error)` — returns hourly completion rate (0-23), peak hours (top 3), slump hours (bottom 3), effort level and mood per hour
- AC4: `StreakAnalysis() (*StreakData, error)` — returns current streak, longest streak, streak history, average streak length, and common streak breakers
- AC5: `BurnoutRisk() (*BurnoutIndicators, error)` — returns composite score (0-1) from: completion trend slope, mood trend, session length trend, bypass rate, task avoidance types, days with no activity. Score >0.7 triggers warning level
- AC6: `WeeklySummary(weekOf time.Time) (*WeeklySummary, error)` — returns velocity, best/worst day, type mix, discovered patterns, and recommendations
- AC7: MCP Resources registered: `threedoors://analytics/mood-correlation`, `threedoors://analytics/time-of-day`, `threedoors://analytics/streaks`, `threedoors://analytics/burnout-risk`, `threedoors://analytics/task-preferences`, `threedoors://analytics/weekly-summary`
- AC8: MCP Tools registered: `get_mood_correlation(from, to)`, `get_productivity_profile(from, to)`, `burnout_risk()`, `get_completions(from, to, group_by, include_mood, include_patterns)`
- AC9: MCP Prompts registered: `daily_summary`, `weekly_retrospective` — template strings guiding LLM to compose multi-tool queries
- AC10: Unit tests for each `PatternMiner` method with synthetic session data

## Technical Notes

- `PatternMiner` builds on the existing `PatternAnalyzer` — expand rather than replace
- Burnout risk is a composite of 5+ signals with configurable weights
- Streak analysis counts consecutive days with at least one completion
- All analytics are computed on-demand from session JSONL data — no pre-computed cache in this story
- Time-of-day analysis uses the user's local timezone inferred from session timestamps
- MCP Prompts are template strings, not executable code — they guide LLMs to call the right tools in sequence

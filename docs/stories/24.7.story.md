# Story 24.7: Task Relationship Graph & Cross-Provider Linking

**Status:** draft
**Epic:** 24 — MCP/LLM Integration Server
**Depends on:** Story 24.2

## User Story

As an LLM client,
I want to traverse task relationship graphs and discover cross-provider dependencies,
So that I can answer questions like "What depends on the auth refactor?" and "What's related across Jira and local tasks?"

## Acceptance Criteria

**Given** tasks exist across multiple providers with potential relationships
**When** an LLM queries graph endpoints
**Then:**

- AC1: `TaskGraph` struct with `Nodes map[string]*TaskNode` and `Edges []TaskEdge` in `internal/mcp/graph.go`
- AC2: `TaskNode` struct: `Task`, `Provider`, `Depth` (distance from query root)
- AC3: `TaskEdge` struct: `FromID`, `ToID`, `Type` (EdgeType), `Weight` (confidence 0-1 for inferred), `Source` ("explicit", "inferred:text", "inferred:temporal")
- AC4: `EdgeType` constants: `blocks`, `related-to`, `subtask-of`, `duplicate-of`, `sequential`, `cross-ref`
- AC5: `RelationshipInferencer` that discovers edges from: text similarity (>70% token overlap → `related-to`), temporal sequences (completed within 30min with similar text → `sequential`), cross-provider refs (shared `SourceRef` → `cross-ref`), blocker chains (blocker text mentions another task → `blocks`), subtask patterns (text subset → `subtask-of`), duplicate detection (via `DuplicateDetector` → `duplicate-of`)
- AC6: MCP Tools: `walk_graph(task_id, depth, direction, edge_types)` → subgraph centered on task, `find_paths(from_id, to_id, max_depth)` → all paths between tasks, `get_critical_path(root_id)` → longest dependency chain, `get_orphans()` → tasks with no relationships, `get_clusters()` → groups of related tasks
- AC7: `CrossProviderLinker` using `MultiSourceAggregator` to discover cross-provider relationships via SourceRef matching, text similarity, temporal proximity, and shared categories
- AC8: MCP Tools: `get_provider_overlap(provider_a, provider_b)`, `get_unified_view(topic)`, `suggest_cross_links()`
- AC9: MCP Resources: `threedoors://graph/dependencies`, `threedoors://graph/cross-provider`
- AC10: Unit tests for graph construction, edge inference, traversal, and cross-provider linking

## Technical Notes

- Graph is computed on-demand, not persisted — edges are inferred from current task state
- Explicit edges come from enrichment DB cross-references; inferred edges are computed at query time
- `walk_graph` uses BFS with configurable depth limit (default 2) and optional edge type filter
- `get_clusters` uses simple connected-component detection, not full community detection algorithms
- Cross-provider linking builds on existing `DuplicateDetector` and `MultiSourceAggregator`
- Inferred edges include confidence weights — LLMs can filter by confidence threshold
- `suggest_cross_links()` returns proposals (via `ProposalStore` from Story 24.4 if available, otherwise raw suggestions)

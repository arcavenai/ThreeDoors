# Story 23.7: Task Edit, Delete, Note, and Search Commands

**Status:** Draft
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.2

## User Story

As a CLI user,
I want to edit task text, delete tasks, add notes, and search tasks from the command line,
So that I have full task management capability without the TUI.

## Acceptance Criteria

**Given** the task list/show commands from Story 23.2 are in place
**When** edit, delete, note, and search commands are implemented
**Then:**

- AC1: `threedoors task edit <id> --text "New text"` updates the task text — supports prefix matching
- AC2: `threedoors task edit <id> --context "Updated context"` updates the task context
- AC3: `threedoors task delete <id>` removes a task — supports prefix matching
- AC4: `threedoors task delete <id1> <id2>` batch deletes multiple tasks
- AC5: `threedoors task note <id> "Progress update"` adds a note to a task
- AC6: `threedoors task search "query"` searches task text and returns matching tasks in list format
- AC7: `threedoors task search "query" --json` outputs search results in JSON envelope with metadata
- AC8: All commands support `--json` flag
- AC9: Delete requires no confirmation in non-interactive mode (for scripting) — the action is explicit
- AC10: Exit code 2 for task not found on edit/delete/note

## Technical Notes

- Edit uses `TaskPool.UpdateTask()` — existing domain method
- Delete uses `TaskPool.RemoveTask()` — existing domain method
- Notes use `Task.AddNote()` — existing domain method
- Search is a simple substring match across task text and context fields
- Reference: `docs/research/cli-interface-design.md` section 4 (Task Management)

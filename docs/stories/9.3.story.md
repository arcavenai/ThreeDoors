# Story 9.3: Performance Benchmarks

**Status:** in-progress

## User Story

As a developer,
I want automated performance benchmarks validating the <100ms NFR,
so that performance regressions are caught before they reach users.

## Acceptance Criteria

1. [x] Benchmark suite using Go's `testing.B` framework
2. [x] Benchmarks for: adapter read, adapter write, adapter sync, door selection algorithm
3. [x] Results compared against <100ms threshold (NFR13)
4. [x] CI integration: benchmarks run on every PR
5. [x] Benchmark results stored for trend analysis (uploaded as artifacts, 90-day retention)

## Implementation Notes

### Benchmark Targets

1. **Adapter Read** — `TextFileProvider.LoadTasks()` with varying task counts (10, 100, 500)
2. **Adapter Write** — `TextFileProvider.SaveTasks()` with varying task counts (10, 100, 500)
3. **Sync Engine** — `SyncEngine.DetectChanges()` and `SyncEngine.Sync()` with realistic task sets
4. **Door Selection** — `SelectDoors()` and `DiversityScore()` with varying pool sizes

### File Layout

- `internal/core/door_selector_bench_test.go` — door selection + diversity score benchmarks
- `internal/core/sync_engine_bench_test.go` — sync engine benchmarks
- `internal/core/task_pool_bench_test.go` — task pool operation benchmarks
- `internal/adapters/textfile/text_file_provider_bench_test.go` — adapter read/write benchmarks
- `Makefile` — `bench` and `bench-save` targets
- `.github/workflows/ci.yml` — `benchmarks` job with NFR13 validation and artifact upload

### Threshold Validation

Each benchmark area includes a `Test*NFR13` function that validates the operation completes
within 100ms for expected workload sizes:
- `TestAdapterReadWriteNFR13` — adapter read/write with 10/100/500 tasks
- `TestSelectDoorsNFR13` — door selection with 10/100/500 pool sizes
- `TestSyncNFR13` — sync operations with 10/100/500 tasks

### CI Integration

The `benchmarks` job in `.github/workflows/ci.yml`:
1. Runs all benchmarks with `-benchmem -count=3` for statistical stability
2. Validates NFR13 thresholds via `Test*NFR13` tests
3. Uploads results as artifacts (90-day retention) for trend analysis

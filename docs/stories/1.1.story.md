# Story 1.1: Project Setup & Basic Bubbletea App

**As a** developer,
**I want** a working Go project with Bubbletea framework,
**so that** I have a foundation for building the Three Doors TUI.

## Acceptance Criteria

1. Go module initialized with `go mod init github.com/arcaven/ThreeDoors`
2. Bubbletea (v1.2.4) and Lipgloss (v1.0.0) dependencies added at pinned versions
3. Basic TUI application renders the header "ThreeDoors - Technical Demo" followed by a blank line and "Press q to quit"
4. Application responds to 'q' and 'ctrl+c' keypresses to quit
5. `Makefile` with `build`, `run`, `clean`, `fmt`, `lint`, and `test` targets works (per `docs/architecture/tech-stack.md`)
6. Application compiles and runs without errors
7. All unit tests pass (`make test`)

## NOT In Scope

*   No domain/business logic (no `internal/tasks/` code)
*   No file I/O, YAML parsing, or task models
*   No session tracking or metrics
*   No additional TUI views beyond the header screen
*   Only the foundational skeleton — future stories build on this

*   All tasks checked off below

*   `make test` passes with zero failures

### Dev Notes

This story establishes the foundational structure of the application. All code should adhere to the project's defined standards.

**Critical coding standards to follow:**
*   Errors must be wrapped with context using `%w` verb
*   No panics in user-facing code — `Update()` and `View()` must never panic
*   Do not use `fmt.Println` for TUI output — all rendering through `View()` only

### Model Specification

The Bubbletea model struct and methods:

```go
// Package tui contains all Bubbletea TUI components.
package tui

// Model is the root Bubbletea model for the application.
// Exported: Model, NewModel()
// Unexported: quitting (accessible within package tui for testing)
type Model struct {
    quitting bool
}

// NewModel creates and returns a new Model instance.
// This is the public constructor used by cmd/threedoors/main.go.
func NewModel() Model { return Model{} }

// Init returns nil (no initial command needed).
func (m Model) Init() tea.Cmd { return nil }

// Update handles key messages: 'q' and ctrl+c trigger tea.Quit.
// All other keys are ignored. Must never panic.
func (m Model) Update(msg tea.Msg) (tea.Model, tea.Cmd) { ... }

// View renders the header and quit hint as PLAIN TEXT (no Lipgloss styling in Story 1.1).
// Exact output: "ThreeDoors - Technical Demo\n\nPress q to quit\n"
func (m Model) View() string { ... }
```

#### Required Test Cases

| Test Name | Validates |
|-----------|-----------|
| `TestModel_Init_ReturnsNil` | `Init()` returns `nil` cmd |
| `TestModel_Update_QuitOnQ` | Pressing 'q' sends `tea.Quit` |
| `TestModel_Update_QuitOnCtrlC` | Pressing ctrl+c sends `tea.Quit` |
| `TestModel_Update_IgnoresOtherKeys` | Other keys don't trigger quit |
| `TestModel_View_ContainsHeader` | Output contains "ThreeDoors - Technical Demo" |
| `TestModel_View_ContainsQuitHint` | Output contains "Press q to quit" |

#### Test Input Patterns

Construct `tea.KeyMsg` for test inputs:
```go
// For 'q' key:
tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune("q")}

// For ctrl+c:
tea.KeyMsg{Type: tea.KeyCtrlC}

// For other keys (e.g., 'a'):
tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune("a")}
```

#### Quit Assertion Pattern

To verify `tea.Quit` was returned:
```go
// Execute the returned cmd and check for tea.QuitMsg
updatedModel, cmd := model.Update(keyMsg)
if cmd == nil {
    t.Fatal("expected quit command, got nil")
}
msg := cmd()  // Execute the command
if _, ok := msg.(tea.QuitMsg); !ok {
    t.Fatalf("expected tea.QuitMsg, got %T", msg)
}
```

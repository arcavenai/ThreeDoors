# Story 3.5.5: Baseline Regression Test Suite

**Status:** in-progress

**As a** developer preparing for Epic 4,
**I want** baseline tests for current door selection and task management,
**So that** the learning engine can be validated against known-good behavior.

## Acceptance Criteria

1. Table-driven tests for random selection, Fisher-Yates, ring buffer, edge cases
2. Status management tests for all state transitions
3. Task pool tests for load, filter, add, remove, update
4. Tests pass on current codebase

## Tasks

1. Write comprehensive door selection regression tests:
   - Fisher-Yates partial shuffle correctness (only first `count` elements shuffled)
   - Diversity-based selection with deterministic seeds
   - Ring buffer wraparound at capacity (10+ marks)
   - Ring buffer eviction (oldest entry replaced)
   - Edge cases: count=0, count=1, count > available, all same categories

2. Write exhaustive status transition tests:
   - Full 7x7 transition matrix (all status pairs)
   - Self-transition no-op behavior
   - CompletedAt timestamp logic for terminal states
   - Blocker clearing on non-blocked transitions
   - Invalid status string handling

3. Write task pool operation tests:
   - Add/Get/Update/Remove lifecycle
   - GetTasksByStatus filtering for all statuses
   - GetAvailableForDoors eligibility rules
   - GetAvailableForDoors fallback when < 3 non-recent tasks
   - Count accuracy through operations
   - UpdateTask overwrites correctly

4. Verify all tests pass on current codebase

# Story 1.2.1: Dynamic Door Height (60% Terminal)

**As a** user,
**I want** doors to fill most of my terminal height,
**so that** the interface feels immersive rather than cramped.

## Acceptance Criteria

1. Doors are rendered at 60% of terminal height.
2. Door height has a minimum of 10 lines to remain usable on small terminals.
3. Door content is vertically centered within each door panel.
4. Doors dynamically resize when the terminal is resized.

## Technical Design

### Door Height Calculation

In `doors_view.go`, door height is computed as:

```go
doorHeight := max(m.height*60/100, 10)
```

This replaces the previous fixed height, making the doors responsive to terminal size.

### Vertical Centering

Content within each door is vertically centered by calculating top padding:

```go
topPadding := (doorHeight - contentHeight) / 2
```

Lines of whitespace are prepended to push content to the vertical center.

### Terminal Size

`main_model.go` stores `tea.WindowSizeMsg` height in the model, which is passed through to the doors view for height calculation.

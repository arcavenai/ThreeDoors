# Story 12.1: Local Calendar Source Reader

## Story

As a user,
I want ThreeDoors to read my local calendar,
So that it understands my available time for task-appropriate door selection.

## Status

In Progress

## Prerequisites

- Epic 4 - COMPLETE (intelligent door selection)

## Acceptance Criteria

**Given** calendar sources configured in config.yaml
**When** the calendar reader initializes
**Then** macOS Calendar.app events are read via AppleScript (no OAuth)
**And** .ics files are parsed from configured paths
**And** CalDAV cache is read from `~/Library/Calendars/`
**And** calendar events are parsed into time blocks (start, end, title)
**And** graceful fallback occurs when calendar sources are unavailable

**Given** a config.yaml with `calendar:` section
**When** the application loads configuration
**Then** calendar sources are enabled/disabled per source type
**And** file paths for .ics and CalDAV cache are configurable

**Given** AppleScript calls for Calendar.app
**When** event data contains special characters in titles or calendar names
**Then** all user/event data is properly escaped

## Technical Design

### Architecture

New `internal/calendar/` package following the architecture docs:

```go
type CalendarReader interface {
    GetEvents(ctx context.Context, start, end time.Time) ([]CalendarEvent, error)
    GetFreeBlocks(ctx context.Context, start, end time.Time) ([]TimeBlock, error)
}
```

### Implementations

1. **AppleScriptCalendarReader** - reads macOS Calendar.app via osascript
2. **ICSCalendarReader** - parses .ics files from configured paths
3. **CalDAVCacheReader** - reads local CalDAV cache from ~/Library/Calendars/
4. **MultiSourceReader** - aggregates events from multiple readers

### New Files

- `internal/calendar/calendar.go` - CalendarEvent, TimeBlock models, CalendarReader interface
- `internal/calendar/applescript_reader.go` - AppleScript-based Calendar.app reader
- `internal/calendar/ics_reader.go` - .ics file parser
- `internal/calendar/caldav_cache_reader.go` - CalDAV cache reader
- `internal/calendar/multi_source_reader.go` - aggregator across sources
- `internal/calendar/config.go` - calendar config types and loading
- `internal/calendar/calendar_test.go` - tests for all readers
- `internal/calendar/testdata/` - test .ics fixtures

### Modified Files

- `internal/tasks/provider_config.go` (updated) - add CalendarConfig field

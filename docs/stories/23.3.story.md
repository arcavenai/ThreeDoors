# Story 23.3: Task Add and Task Complete Commands

**Status:** Draft
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.1

## User Story

As a CLI user,
I want to add new tasks and mark tasks complete from the command line,
So that I can manage my task lifecycle without the TUI.

## Acceptance Criteria

**Given** the CLI scaffold from Story 23.1 is in place
**When** the task add and complete commands are implemented
**Then:**

- AC1: `threedoors task add "Task text"` creates a new task with the given text and status `todo`
- AC2: `threedoors task add "Task text" --context "Why this matters"` creates a task with context
- AC3: `threedoors task add "Task text" --type code --effort medium` creates a task with type and effort metadata
- AC4: `threedoors task add --json` outputs the created task in JSON envelope format
- AC5: Human-readable output confirms creation: "Created task <short-id>: <text>"
- AC6: `threedoors task complete <id>` marks a task as complete — supports prefix matching
- AC7: `threedoors task complete <id1> <id2> <id3>` batch completes multiple tasks (variadic args)
- AC8: Batch complete processes all items; exits 0 if all succeed, exits 1 if any fail
- AC9: `threedoors task complete --json` outputs per-item results in JSON
- AC10: Completion records timestamp via existing `UpdateStatus` flow
- AC11: Exit code 2 if task ID not found, exit code 3 if invalid status transition

## Technical Notes

- Task creation uses `TaskPool.AddTask()` — the domain logic already handles ID generation and validation
- Batch operations should collect results and report all, not fail-fast on the first error
- Prefix matching reuses `FindByPrefix` from Story 23.2 (or can be implemented independently if 23.3 ships first)
- Reference: `docs/research/cli-interface-design.md` section 4 (Task Management)

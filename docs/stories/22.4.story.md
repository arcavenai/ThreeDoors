# Story 22.4: Dev Queue View (List, Approve, Kill)

**Status:** Done (PR #TBD)
**Epic:** 22 — Self-Driving Development Pipeline
**Depends on:** Stories 22.1, 22.2

## User Story

As a user,
I want a dev queue view where I can see pending dispatches, approve them, and kill running workers,
So that I maintain control over what gets dispatched and can stop runaway agents.

## Acceptance Criteria

**Given** the user opens the dev queue view via `:devqueue` command
**When** the view renders
**Then:**

- AC1: Queue items displayed as a scrollable list with columns: Status (icon), Task Text (truncated to fit), Worker Name, PR # (if available), Queued At (relative time)
- AC2: 'y' key on a pending item triggers `Dispatcher.CreateWorker` with the queue item's task description, updates status to `dispatched`
- AC3: 'n' key on a pending item removes it from the queue (with brief confirmation)
- AC4: 'K' key on a dispatched/running item calls `Dispatcher.RemoveWorker` and updates status to `failed` with error "Killed by user"
- AC5: 'j'/'k' or up/down arrow keys navigate the list; current item highlighted
- AC6: ESC returns to the previous view (doors or detail)
- AC7: Status icons rendered via Lipgloss: ⏳ pending, ⚙️ dispatched, ✅ completed, ❌ failed
- AC8: Empty state message: "No items in dev queue. Dispatch a task with 'x' from detail view."
- AC9: Worker creation wrapped in `tea.Cmd` — no blocking I/O in Update()

# Story 8.3: Obsidian Vault Configuration

Status: in-progress

## Story

**As a** user,
**I want** to configure my Obsidian vault path, target folder, and file naming via config.yaml,
**so that** ThreeDoors integrates with my specific vault structure.

## Acceptance Criteria

1. **Config.yaml Obsidian Section** (AC:1)
   - Given a config.yaml with `obsidian:` provider entry
   - When the application starts
   - Then `vault_path`, `tasks_folder`, and `file_pattern` settings are respected

2. **Vault Path Validation** (AC:2)
   - Given an obsidian provider is configured
   - When the application starts
   - Then the vault path is validated (exists, is directory, readable, writable)

3. **Invalid Vault Path Fallback** (AC:3)
   - Given an invalid vault path in config
   - When the application starts
   - Then a clear error message is produced
   - And the system falls back to the text file provider

4. **Default Task Folder** (AC:4)
   - Given no `tasks_folder` setting
   - When the adapter is initialized
   - Then the vault root is used as the task directory

5. **File Pattern Glob** (AC:5)
   - Given a `file_pattern` setting (e.g., `*.md`, `tasks/*.md`)
   - When the adapter loads tasks
   - Then only files matching the glob pattern are scanned

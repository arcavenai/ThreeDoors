# Story 22.7: Optional Story File Generation via AgentService

**Status:** draft
**Epic:** 22 — Self-Driving Development Pipeline
**Depends on:** Story 22.2

## User Story

As a user,
I want to optionally generate story files before dispatching a task,
So that workers receive structured requirements following the project's story-driven development pattern.

## Acceptance Criteria

**Given** `require_story: true` is set in the dev dispatch config section
**When** a task is approved for dispatch (from dev queue view)
**Then:**

- AC1: Before spawning the worker, `AgentService.DecomposeAndWrite()` is called to generate BMAD-style story files from the task text and context
- AC2: Generated story files are committed to a feature branch before the worker is spawned
- AC3: Worker task description includes instructions to implement the generated stories, referencing the branch and story file paths
- AC4: If `require_story: false` (the default), story generation is skipped entirely and the worker receives the raw task description
- AC5: Story generation failure is non-fatal — the error is logged, a warning is set on the queue item (`DispatchErr: "story generation failed: <reason>"`), and dispatch proceeds with the raw task description
- AC6: `require_story` defaults to `false` in the dev dispatch config section
- AC7: Story generation is wrapped in a `tea.Cmd` — it must not block the TUI update loop

## Technical Notes

- The existing `AgentService` is in `internal/intelligence/agent_service.go` — it already has `DecomposeAndWrite()` which uses an LLM to break tasks into story specs and write them to git
- This story integrates the existing capability, not reimplements it
- The dispatch engine needs a reference to `AgentService` — inject via constructor or optional dependency
- If `AgentService` is not configured (no LLM backend), treat `require_story: true` as a soft failure (log warning, proceed without stories)
- Two-phase dispatch: (1) generate stories → commit to branch, (2) spawn worker pointing at the branch
- Branch naming for story commits: `stories/<queue-item-id>` or similar

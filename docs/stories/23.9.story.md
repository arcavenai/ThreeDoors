# Story 23.9: Config Commands and Stdin/Pipe Support

**Status:** Draft
**Epic:** 23 — CLI Interface
**Depends on:** Story 23.1, Story 23.3

## User Story

As a CLI user or LLM agent,
I want to view/modify configuration and pipe task text via stdin,
So that I can script task creation and configure ThreeDoors without editing files manually.

## Acceptance Criteria

**Given** the CLI scaffold and task add command are in place
**When** config commands and stdin support are implemented
**Then:**

- AC1: `threedoors config show` displays the full configuration in human-readable format
- AC2: `threedoors config show --json` outputs config as JSON envelope
- AC3: `threedoors config get <key>` outputs a single config value
- AC4: `threedoors config set <key> <value>` updates a config value and persists to config file
- AC5: `echo "Buy groceries" | threedoors task add` reads task text from stdin when not a TTY
- AC6: `cat tasks.txt | threedoors task add --stdin` reads multiple tasks (one per line) from stdin
- AC7: Stdin detection uses `term.IsTerminal(int(os.Stdin.Fd()))` — auto-detect, no flag required for single task
- AC8: `--stdin` flag explicitly enables multi-line stdin mode (one task per line)
- AC9: Config key validation — unknown keys return exit code 3
- AC10: Pipe-created tasks output their IDs (one per line) for downstream command chaining

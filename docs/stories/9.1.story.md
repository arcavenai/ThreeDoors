## Implementation Notes

### Test Structure

The E2E test suite is organized into 6 test categories (31 tests total):

- **Note Creation** (1 test): New task append via read-modify-write cycle
- **Task Read** (6 tests): Fixture loading, special chars, large notes, mixed formats, empty notes, deterministic IDs
- **Task Update** (3 tests): Toggle completion, batch update, delete task
- **Bidirectional Sync** (3 tests): Read-modify-write cycle, multiple round trips, ID stability
- **Error Handling** (8 tests): Note not found, permission denied, timeout, read-only, write failures, read failures
- **Connectivity Failure** (3 tests): osascript not found, intermittent timeout, FallbackProvider integration
- **Partial Sync** (3 tests): Write fails after read, batch partial failure, delete during modification
- **Concurrent Modification** (4 tests): Parallel reads, parallel writes, read during write, parallel deletes

### Contract Test Subset

Additionally, `internal/adapters/applenotes_contract_test.go` provides a contract-compatible test suite using an in-memory note store simulation. The full `RunContractTests` suite is NOT applicable because AppleNotesProvider uses position-based SHA-1 IDs that don't round-trip with `NewTask()` UUIDs. The subset tests validate: save/load text preservation, append, update by position ID, delete by position ID, batch operations, and concurrent access.

### Test Fixtures

5 fixture files in `testdata/applenotes/`:
- `basic_tasks.txt` — standard checkbox tasks
- `empty_note.txt` — empty note edge case
- `special_chars.txt` — HTML entities, unicode, quotes
- `large_note.txt` — 20-line multi-section note
- `mixed_formats.txt` — dash/asterisk/capital-X/indented formats

### Key Design Decision

All tests use `CommandExecutor` mock injection via `NewAppleNotesProviderWithExecutor()` — no real Apple Notes or osascript needed for CI.
